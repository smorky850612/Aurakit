# EXPLAIN 모드 — 코드 이해 및 설명

> 코드를 작성하는 게 아니라 **이해하는** 모드. REVIEW(감사)와 다름.

## 트리거
```bash
/aura explain: [대상]
/aura explain:src/auth/login.ts
/aura explain: 이 에러가 왜 발생해?
```

## 파이프라인

1. **Scout 탐색** — 대상 파일/함수/에러 파악
2. **컨텍스트 수집** — 관련 파일, 호출 체인, 의존성
3. **설명 생성** — 사용자 레벨에 맞게 (style:learning 연동)
   - 초보자: 비유 + 단계별 설명
   - 중급자: 코드 흐름 + 패턴 설명
   - 전문가: 아키텍처 결정 + 트레이드오프
4. **다이어그램** (선택) — 흐름도, 의존성 트리

## 출력 형식
- 한 문단 요약 (TL;DR)
- 상세 설명 (단계별)
- 관련 코드 인용
- "더 알아보기" 링크 (해당 모드 전환 제안)

## 다음 추천

- 리팩토링 필요 → `/aura clean:`
