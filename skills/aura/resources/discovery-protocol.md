# AuraKit — Discovery Protocol

> BUILD 모드 Step 1 필수 실행. 5-point 체크리스트. 구현 전 반드시 완료.

---

## Discovery 5-체크

| # | 체크 | 질문 |
|---|------|------|
| D-1 | 실제 문제 | "이 기능이 해결하는 실제 문제는?" |
| D-2 | 기존 코드 | "이미 비슷한 코드가 있는가?" (grep 탐색 필수) |
| D-3 | YAGNI | "지금 당장 필요한가, 아니면 미래를 위한 것인가?" |
| D-4 | 범위 | "무엇을 포함하고 무엇을 제외할 것인가?" |
| D-5 | 성공 기준 | "어떻게 완료를 알 수 있는가?" |

---

## Discovery 실행 (Scout, haiku, context:fork)

```bash
# 기존 코드 탐색
grep -r "[키워드]" src/ --include="*.ts" -l
find src/ -name "*[관련파일]*" 2>/dev/null

# 의존성 확인
cat package.json | grep -E '"[관련 패키지]"'

# 관련 API 확인
grep -r "export.*function\|export default" src/ --include="*.ts" -l | head -20
```

## 출력 포맷 (200 토큰 이내)

```
🔍 Discovery — [기능명]

D-1 문제: [실제 해결 문제]
D-2 기존 코드: [있음/없음] — [경로 또는 "새로 생성"]
D-3 YAGNI: [지금 필요 / 나중에]
D-4 범위: 포함 [항목] / 제외 [항목]
D-5 성공 기준: [검증 방법]

→ [구현 진행 / 기존 코드 활용 / 범위 조정 필요]
```

---

## YAGNI 판단 기준

- ✅ 지금 필요: 사용자 요청, 버그 수정, 성능 블로커
- ❌ YAGNI: "나중에 필요할 수도", "확장성을 위해", "미래 대비"

---

## Discovery 결과별 대응

| 결과 | 대응 |
|------|------|
| 기존 코드 있음 | 재사용 or 리팩토링 → 중복 구현 금지 |
| YAGNI 해당 | 사용자에게 범위 조정 제안 |
| 범위 불명확 | 마이크로 플랜 전 추가 질문 |
| 성공 기준 없음 | 사용자에게 검증 방법 명시 요청 |
